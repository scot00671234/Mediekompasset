Metodologi

En detaljeret gennemgang af vores tekniske tilgang til medieanalyse

Teknisk implementation

Vores analyse er baseret på en kombination af avancerede Natural Language Processing (NLP) teknikker og maskinlæring, specifikt tilpasset til det danske sprog.

Dataindsamling

  • Automatiseret web scraping med Python
  • Selenium WebDriver for dynamisk indhold
  • Asynkron dataindsamling med aiohttp
  • BeautifulSoup4 for HTML parsing

Sprogprocessering

  • SpaCy med dansk sprogmodel
  • Danish BERT (Maltehb/danish-bert-botxo)
  • TF-IDF vektorisering
  • Custom tokenization

Analyse framework

  • FastAPI backend
  • Jinja2 templating
  • Pandas for databehandling
  • scikit-learn for ML

Detaljeret teknisk beskrivelse

Dataindsamling og preprocessing

Vores scraper-implementation bruger følgende teknologier:

  • Asynkron scraping med aiohttp for effektiv dataindsamling
  • Selenium WebDriver for JavaScript-renderet indhold
  • Custom User-Agent rotation og rate limiting
  • Automatisk fejlhåndtering og retry-mekanismer
  • HTML cleaning og tekstnormalisering

Natural Language Processing

Vores NLP-pipeline består af følgende komponenter:

  • SpaCy's danske sprogmodel (da_core_news_sm) for grundlæggende sproganalyse
  • Danish BERT model fra BotXO for kontekstuel analyse
  • Custom politisk ordbog med vægtede nøgleord
  • TF-IDF vektorisering for emnemodellering
  • K-means clustering for emnegrupering

Bias-Scoring System

Vores bias-vurdering kombinerer flere analytiske tilgange:

  • Regelbaseret analyse baseret på vægtede politiske nøgleord
  • BERT-baseret sentimentanalyse tilpasset dansk politik
  • Hybrid scoring der kombinerer regelbaseret og ML-baseret vurdering
  • Konfidensberegning baseret på tekstmængde og nøgleordforekomster

Tekniske metrikker

Vi beregner følgende kvantitative metrikker:

  • Politisk bias: Normaliseret score mellem -1 og 1
  • Faktualitet: Baseret på kildehenvisninger og verificerbare udsagn
  • Kildediversitet: Shannon diversitetsindex af citerede kilder
  • Emnebalance: Entropi-baseret måling af emnedistribution

Tekniske begrænsninger

Som med alle automatiserede analysesystemer har vores implementation nogle iboende begrænsninger:

  • BERT-modellen er begrænset til 512 tokens per tekst-segment
  • Sprogmodellen kan have udfordringer med nye politiske termer
  • Ironi og sarkasme er særligt udfordrende at detektere automatisk
  • Kontekstafhængige politiske referencer kræver løbende opdatering af ordbogen
  • Bias-scoring er baseret på eksplicitte sproglige markører og kan overse subtile bias
